# 高端民宿度假区 Guest Portal 产品需求文档（PRD）

## 1. 产品概述
- 面向高端民宿度假区住客的统一数字入口，贯穿住前、住中、住后全旅程。
- 设备覆盖：微信/支付宝小程序、移动 H5、房内平板、房间电视投屏；原生 App 作为后续规划。
- 价值主张：以最少步骤完成核心任务（在线入住、开门、预约/点餐、工单、结算），并通过礼宾与个性化推荐提升二次消费与满意度。

## 2. 业务目标与指标
- 满意度：NPS ≥ 70；IM 礼宾满意度 ≥ 4.7/5；客诉率 ≤ 0.5%。
- 转化与履约：住中二次消费转化率 ≥ 18%；工单履约准时率 ≥ 95%。
- 关键链路性能：开门成功率 ≥ 99.5%；关键操作 P95 ≤ 500ms。
- 结算与对账：账单差异率 ≤ 0.1%；退款处理时长 ≤ T+3。

## 3. 用户画像与旅程
- 高净值旅客：注重私密与定制化服务；偏好专属礼宾与定制活动。
- 家庭度假：关注亲子设施、餐饮预约与活动安排；需要多人分账与行程共享。
- 团队/婚礼包场：强调日程管理、场地协调、清晰账单与发票。
- 园区员工（B 端）：前台、礼宾、客房、餐饮、运维与安保。

### 全旅程
- 住前：确认预订、偏好收集、在线入住（证件、人脸、押金）、到达指引。
- 住中：数字钥匙开门、客房服务与报修、餐饮预约和点餐、活动报名、礼宾对话、园区导览。
- 住后：账单结算、发票开具、会员权益沉淀、评价与复购激励。

## 4. 信息架构（IA）
- 主页：欢迎与房态、数字钥匙入口、当日推荐与快捷操作。
- 我的行程：预订详情、入住/退房流程、账单与发票、行程共享。
- 服务：客房服务与工单、报修、清洁与夜床、接送机与行李。
- 餐饮&活动：餐厅预约、菜单点餐、活动报名与候补、定制行程。
- 消息：系统通知、礼宾会话、订单状态更新、紧急广播。
- 个人中心：会员等级与权益、偏好与隐私、家庭/团队成员管理。

## 5. 详细功能需求与验收
### 5.1 预订与在线入住
- 住前信息收集：到达时间、偏好（枕头/过敏原/楼层）、特殊需求。
- 身份校验：证件 OCR + 人脸核验；押金支付；协议签署；异常兜底人工。
- 验收：全流程 ≤ 5 分钟；失败兜底人工响应 ≤ 1 分钟。

### 5.2 数字钥匙与门禁
- 凭证：`NFC`、`BLE`、二维码动态令牌；房间与公共区授权；离线容错。
- IoT 联动：进房自动场景（空调、灯光、窗帘）；亲子模式与安防提醒。
- 验收：开门成功率 ≥ 99.5%；平均解锁时延 ≤ 500ms；异常即时通知。

### 5.3 餐饮与客房服务
- 菜单浏览、过敏原过滤、到店/送餐、时段与等候管理。
- 客房服务一键下单（加床品、清洁、夜床）；工单状态与 SLA 跟踪。
- 验收：下单至接单 ≤ 60 秒；履约准时率 ≥ 95%；客诉自动升级礼宾。

### 5.4 活动与礼宾
- 活动日程（瑜伽、徒步、品酒等）：报名/候补、容量控制、天气联动。
- 私人礼宾：1:1 会话、行程规划、打包服务与支付；消息既读回执。
- 验收：活动满员率 ≥ 80%；礼宾响应 ≤ 1 分钟；满意度 ≥ 4.7/5。

### 5.5 账单与结算
- 订单统一视图、分账（家庭/团队）、押金管理、发票抬头。
- 支付：微信/支付宝、`Apple Pay`、国际信用卡（含 `3DS`）；离线后补结算。
- 验收：账单差异率 ≤ 0.1%；退款时长 ≤ T+3；对账自动化 ≥ 95%。

### 5.6 会员与权益
- 等级与积分、连住礼遇、生日/周年特典、联名权益。
- 营销：住中加购券、满减包、动态权益推荐与黑名单防滥用。
- 验收：权益核销成功率 ≥ 99%；复购率提升 ≥ 12%。

### 5.7 多语言与本地化
- 中/英/日；货币、度量、时区与节假日规则本地化。
- 验收：语言切换无丢字；菜单与活动多语言覆盖 ≥ 95%。

### 5.8 通知与消息
- 模板化推送（入住指引、订单更新、紧急广播）、礼宾 IM、既读与送达回执。
- 验收：通知送达率 ≥ 98%；关键事件去重正确率 ≥ 99%。

## 6. 运营后台（B 端）
- 住客管理：行程、偏好、风险与黑名单；家庭/团队关系。
- 服务编排：餐饮与活动上架、容量与时段、价格与库存策略。
- 工单与履约：服务分发、SLA 配置、异常升级路径与数据面板。
- 内容与营销：模板、权益、券包、AB 实验、人群运营。
- 报表与对账：订单、退款、发票、结算与差异分析。

## 7. 外部系统集成
- PMS/CRS：预订、房态、住客档案；双向同步、冲突解决策略（乐观锁与最新写入策略）。
- 门禁与 IoT：`BLE/NFC/QR`、设备状态与场景编排；离线兜底策略（本地缓存+延迟同步）。
- 支付与发票：微信/支付宝、国际卡与 `3DS`、增值税发票开具与查重。
- 工单与客服：服务排队、路由与优先级；IM 与机器人协作（FAQ 与意图识别）。

## 8. 数据模型（简要）
- 实体：`Guest`、`Reservation`、`Stay`、`Order`、`ServiceRequest`、`Activity`、`Bill`、`Membership`、`AccessGrant`。
- 关系：`Guest`↔`Reservation`(1..n)、`Stay`↔`AccessGrant`(1..n)、`Stay`↔`Order`(1..n)。
- 隐私：最小必要与目的限定；住后匿名化与可删除；留存策略遵从当地法规。

## 9. 权限与安全
- 认证：`OAuth2/OIDC` + `PKCE`；入住强认证（证件+人脸）。
- 授权：`RBAC`（Guest/FamilyAdmin/CorporateAdmin/Staff）+ 细粒度资源授权（房间、公共区）。
- 安全：防重放与动态密钥、传输加密、支付 `PCI DSS`、操作审计与异常告警。

## 10. 性能与 SLA
- 门禁与关键操作 P95 ≤ 500ms；峰值并发 5k RPS；海外与园区双接入。
- 可靠性：关键链路降级与熔断、离线本地缓存与延迟同步；灰度与回滚策略。

## 11. 可访问性与合规
- `WCAG 2.1 AA`：色彩对比、语义化标签、键盘可达、屏幕阅读。
- 合规：当地旅业法规、未成年人保护、跨境数据与税务/发票规范。

## 12. 可观测性与分析（埋点与指标）
### 12.1 埋点事件
- `reservation_view`、`checkin_start`、`checkin_success`、`checkin_fail`。
- `door_unlock_attempt`、`door_unlock_success`、`door_unlock_fail`、`iot_scene_apply`。
- `fnb_menu_view`、`fnb_order_submit`、`fnb_order_paid`、`fnb_order_cancel`。
- `service_request_create`、`service_request_accept`、`service_request_sla_breach`。
- `activity_list_view`、`activity_register`、`activity_waitlist_join`、`activity_attend`。
- `concierge_open_chat`、`concierge_msg_send`、`concierge_csat_submit`。
- `bill_view`、`payment_start`、`payment_success`、`refund_request`。

### 12.2 指标口径
- 转化漏斗：住前→入住→住中消费→结算；各环节转化率与时长分布。
- SLA：接单时长、履约准时率、异常闭环率；消息送达率与重复去重。
- 质量：开门成功率、IoT 场景应用成功率、支付成功率、账单差异率。
- 运营：活动满员率、券包核销率、权益使用率、复购率与留存。

### 12.3 埋点字段规范
- 公共字段：`user_id`/`stay_id`/`reservation_id`、`locale`、`app_version`、`device_type`、`network_type`、`timezone`、`trace_id`。
- 业务字段示例：
  - `door_unlock_*`：`room_no`、`mode(BLE/NFC/QR)`、`latency_ms`、`retry_count`、`error_code`。
  - `fnb_order_*`：`restaurant_id`、`items_count`、`allergen_flags`、`eta_min`、`payment_method`。
  - `service_request_*`：`type`、`slot`、`sla_min`、`route_department`、`status`。
- 隐私与采集：最小必要、可撤回与删除；PII 加密与脱敏；跨境数据合规。

### 12.4 数据看板与告警
- 实时：关键链路成功率（入住、开门、支付）、异常闭环率；P95 时延。
- 每日：转化漏斗、活动与餐饮运营、SLA 达标率与客诉分析。
- 告警：门禁成功率 < 98.5%、支付成功率突降、工单 SLA 违约率 > 5%、通知送达率 < 96%。

## 13. 用户流程（关键路径）
### 13.1 在线入住（住前→入住）
1. 住客打开 Portal → 查看预订详情 → 开始入住。
2. 上传证件（OCR）→ 人脸核验 → 确认到达时间与偏好 → 支付押金。
3. 同意协议 → 成功后生成 `AccessGrant` → 进入主页显示数字钥匙入口。
4. 失败兜底：跳转人工协助，礼宾 1 分钟内响应。

### 13.2 开门与场景联动（住中）
1. 主页点击数字钥匙 → 选择房间 → 近场触发（`BLE/NFC`）或二维码扫描。
2. 验证令牌 → 解锁成功 → 应用 IoT 入房场景；若失败，展示离线容错与提示。

### 13.3 餐饮预约与点餐
1. 浏览餐厅与时段 → 选择就餐人数 → 预约成功或进入等候队列。
2. 菜单查看（过敏原过滤）→ 下单 → 实时状态跟踪 → 餐厅接单与备餐。

### 13.4 客房服务与工单
1. 服务列表选择（加床品/清洁/报修）→ 填写备注与时段 → 下单。
2. 工单路由到对应部门 → 接单 → 履约 → 住客评价或升级礼宾。

### 13.5 结账与发票（退房）
1. 查看账单（房费+餐饮+活动+服务）→ 分账或合并 → 支付完成。
2. 开具发票（增值税抬头）→ 发送至邮箱/微信卡包。

### 13.6 文本线框（关键页面元素）
- 主页
  - 顶部：欢迎语与房态标签（已入住/未入住/退房倒计时）。
  - 中部：数字钥匙卡片（状态、授权范围、异常提示与重试按钮）。
  - 快捷：餐饮预约、活动报名、客房服务卡片（带状态徽标）。
  - 底部：消息预览与礼宾一键入口。
- 我的行程
  - 预订信息卡：日期、房型、人数、订单号；入住流程进度条（证件→押金→协议→钥匙）。
  - 账单总览卡：当前消费合计、待支付、分账入口；行程共享二维码。
- 服务
  - 分类列表：客房/报修/清洁/夜床/交通；每项显示 SLA 与可预约时段。
  - 详情页：备注输入、时段选择、价格与加购；下单按钮与预计响应时间。
- 餐饮&活动
  - 餐厅卡：营业时段、席位剩余、评分与菜系标签；预约按钮。
  - 菜单页：菜品卡（图片、价格、过敏原标签、可选配料）；购物车与到店/送餐选择。
  - 活动卡：时间、容量、难度、天气提示；报名/候补按钮与规则说明。
- 消息
  - 列表：系统通知与礼宾会话分组；紧急广播置顶红标。
  - 会话：文本/图片/位置；既读状态；常用快捷短语与服务入口。
- 个人中心
  - 会员卡：等级、积分、权益；近三月住次与里程碑。
  - 偏好：过敏原、枕头类型、打扫频次、夜床；隐私设置与数据下载。
  - 家庭/团队：成员管理、权限（分账/开门）、关系绑定与共享行程。

## 15. 接口规约草案（示例）
### 15.1 认证
- `POST /auth/oauth/token`：`PKCE` 授权；返回 `access_token` 与 `id_token`。
- `POST /auth/checkin/verify`：证件 OCR 与人脸比对。

示例：

请求（`POST /auth/oauth/token`）
```json
{
  "grant_type": "authorization_code",
  "code": "abc123",
  "code_verifier": "verifier",
  "redirect_uri": "https://portal.example.com/callback"
}
```

响应
```json
{
  "access_token": "ey...",
  "id_token": "ey...",
  "token_type": "Bearer",
  "expires_in": 3600
}
```

### 15.2 预订与入住
- `GET /reservations/{id}`：获取预订详情。
- `POST /checkin`：提交入住信息与押金支付；返回 `stay_id` 与 `access_grant_id`。

请求（`POST /checkin`）
```json
{
  "reservation_id": "R12345",
  "guest": {
    "name": "张三",
    "id_no": "1101**********",
    "face_token": "ftk..."
  },
  "arrival": {
    "eta": "2025-12-01T18:30:00+08:00",
    "preferences": {"pillow": "soft", "allergen": ["peanut"]}
  },
  "deposit": {"amount": 2000, "currency": "CNY", "method": "wechat"}
}
```

响应
```json
{
  "stay_id": "S67890",
  "access_grant_id": "AG999",
  "status": "confirmed"
}
```

### 15.3 门禁与 IoT
- `POST /access/grant`：创建房间与公共区授权；返回动态令牌。
- `POST /access/unlock`：近场/二维码解锁；返回状态与时延；失败码与兜底建议。
- `POST /iot/scene/apply`：应用场景（入房/夜间/亲子）；返回设备执行结果。

请求（`POST /access/unlock`）
```json
{
  "grant_id": "AG999",
  "room_no": "A-302",
  "mode": "BLE",
  "nonce": "n-123",
  "idempotency_key": "unlock-A302-20251119-1"
}
```

响应
```json
{
  "status": "success",
  "latency_ms": 312,
  "applied_scene": "checkin_welcome",
  "trace_id": "t-xyz"
}
```

### 15.8 错误码与幂等
- 错误码分层：认证/授权/业务/外部依赖；`429` 节流与重试。
- 幂等键：支付、开门、工单创建；`Idempotency-Key` + 过期策略。

统一错误响应示例
```json
{
  "error": {
```
